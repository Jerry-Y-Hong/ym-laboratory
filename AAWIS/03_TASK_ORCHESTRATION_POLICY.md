## 1. Task Delegation Policy

Tasks generated by the Workflow Execution Engine must be orchestrated down to AAOS with strict delegation rules:
- **Least Privilege Execution**: Tasks must request only the minimum required AAOS agent permissions.
- **Resource Bounding**: Each task is assigned a specific Timeout (TTL) and Compute Quota.

---

## 2. Concurrency & Synchronization

- **Parallel Execution**: Independent task nodes in the workflow DAG must be executed concurrently to optimize latency.
- **Barrier Synchronization**: Workflows must wait for all parallel branch dependencies to resolve before proceeding to a unified downstream node.

---

## 3. Human-in-the-Loop (HITL) Orchestration

- Tasks flagged with `requires_human_approval: true` will pause the execution thread.
- The state is serialized and persisted.
- Execution resumes only upon receiving cryptographically signed Project Owner approval.
