# 📐 대체 식재료 및 약재 시뮬레이터 기술 사양서
## (Ingredient Substitution Simulator Technical Specification)

본 문서는 MFCO 플랫폼의 핵심 차별화 포인트(USP)인 **"대체 식재료 및 약재 시뮬레이터"**의 수학적 모델, 데이터 아키텍처 및 인터페이스 사양을 정의합니다. 본 사양서를 바탕으로 향후 개발팀이 즉각적인 기능 구현에 착수할 수 있도록 구체적인 로직을 수록합니다.

---

## 1. 개요 및 비즈니스 가치 (Unique Selling Proposition)

기존의 단순 건강식 추천 시스템과 달리, MFCO 대체 시뮬레이터는 **약선 요리 전문가, 프랜차이즈 R&D 메뉴 기획자, 셰프**들이 기존 요리를 베이스로 삼아 **특정 건강 상태에 커스텀 마이징된 새로운 약선 요리를 발명하도록 돕는 인공지능 기반 창작 도구**입니다.

### 핵심 가치
* **전문가 협업 시너지**: 한의학적 기미론과 현대 영양학 데이터를 결합한 실시간 시뮬레이션을 제공하여 연구 개발(R&D) 시간을 획득합니다.
* **메뉴 다각화 및 프랜차이즈화**: 하나의 베이스 메뉴(예: 삼계탕)에서 '기력 강화 삼계탕', '안심 소화 삼계탕', '순환 온기 삼계탕' 등 타겟 고객 상태별 파생 메뉴 개발을 자동화합니다.

## 2. 수학적 대체 적합도 모델 (Mathematical Modeling)

특정 레시피에서 식재료 $I_{old}$를 새로운 식재료 $I_{new}$로 대체할 때, 시스템은 아래의 다차원 연산 공식을 통해 대체 적합도를 평가합니다.

$$\text{대체 적합도 Score}(I_{new}) = w_1 \cdot \text{EfficacySim}(I_{old}, I_{new}) + w_2 \cdot \text{PropertyMatch}(I_{new}, R_{base}) - w_3 \cdot \text{TasteDissonance}(I_{new})$$

*가중치 기준: $w_1 = 0.5$, $w_2 = 0.3$, $w_3 = 0.2$ (설정 및 튜닝 가능)*

### 1) 효능 유사도 연산: $\text{EfficacySim}(I_{old}, I_{new})$
각 식재료의 25대 생리 기능(SF001~SF025) 매핑 강도를 차원 벡터 $\vec{V}$로 표현하고, 코사인 유사도(Cosine Similarity)를 통해 기능적 유사성을 계산합니다.

$$\text{EfficacySim}(I_{old}, I_{new}) = \frac{\vec{V}_{old} \cdot \vec{V}_{new}}{\|\vec{V}_{old}\| \|\vec{V}_{new}\|}$$

* **유사도가 높을 때 (1.0에 수렴)**: 기존 요리의 치료/웰니스 목적을 완벽하게 대체할 수 있음을 의미 (예: 황기 ↔ 인삼).
* **유사도가 낮을 때 (0.0에 수렴)**: 완전히 새로운 효능을 요리에 주입함을 의미 (이 경우 목적 변경 가이드 제공).

### 2) 성질(氣味) 정합성 연산: $\text{PropertyMatch}(I_{new}, R_{base})$
새로운 식재료의 사기(四氣: 한, 량, 평, 온, 열) 및 오행 속성이 기존 레시피의 메인 에너지 성향과 조화를 이루는지 연산합니다.
* 예: 찬 성질의 오리탕 레시피에 또다시 극찬(極寒) 성질의 약재를 대체 투입할 경우 정합성 페널티 부여.
* 사기(四氣) 열역학 지수 매핑: `열(Hot)` = +2, `온(Warm)` = +1, `평(Neutral)` = 0, `량(Cool)` = -1, `한(Cold)` = -2.
$$\text{PropertyMatch} = 1.0 - \frac{|\text{TempIndex}(I_{new}) - \text{MeanTempIndex}(R_{base})|}{4}$$

### 3) 미각 이질성 연산: $\text{TasteDissonance}(I_{new})$
식재료의 오미(五味: 산, 고, 감, 신, 함) 속성이 결합했을 때, 미각적으로 불쾌감(예: 과도한 쓴맛 또는 지나친 신맛)을 유발하는지 감점 요소를 계산합니다.
* 쓴맛(`고味`)이 강한 약재 대체 투입 시 **-0.5 페널티**를 기본 적용하되, 감초/대추 등 단맛(`감味`)의 보완 약재가 레시피 내에 존재할 경우 페널티 상쇄(Mitigation) 처리.

## 3. 대체 매커니즘 상세 데이터 시나리오

삼계탕 레시피에서 **인삼(H0002) ➡️ 산약(H0005: 마)**으로 대체 시뮬레이션을 돌렸을 때의 백엔드 연동 데이터 예시입니다.

### Step 1. 원본 및 대체 식재료 온톨로지 정보 로드
* **인삼 (H0002)**
  * 오행/성질/맛: `토 (Earth)` / `온 (Warm)` / `감·미고 (Sweet, Slightly Bitter)`
  * 매핑 기능: `PRIMARY_SF: SF012` (체력회복/보기), `SECONDARY_SF: SF022` (건비/비위강화)
* **산약 (H0005)**
  * 오행/성질/맛: `토 (Earth)` / `평 (Neutral)` / `감 (Sweet)`
  * 매핑 기능: `PRIMARY_SF: SF022` (건비/비위강화), `SECONDARY_SF: SF021` (영양흡수)

### Step 2. 대체 시뮬레이션 연산
1. **효능 벡터 변화 분석**:
   * 인삼의 `SF012`가 제외되므로 **보기(Energy Boost) 기능은 -30% 감소**하나, 산약의 `SF021`이 추가되므로 **영양흡수 및 소화 보호 기능은 +40% 증가**함.
2. **열역학 성질 변화 분석**:
   * 인삼의 `온(Warm, +1)` 성질이 산약의 `평(Neutral, 0)` 성질로 변경됨.
   * "성질이 평이해짐에 따라, 몸에 열이 많은 사람(소양인 등)도 부작용 없이 장기 섭취가 가능한 삼계탕 레시피로 성향이 전환됩니다."
3. **미각 변화 분석**:
   * 인삼의 `미고(Slightly Bitter)` 맛이 산약의 끈끈하고 부드러운 `단맛(Sweet)`으로 대체되어 불쾌한 약재 향이 감소하고 국물이 한층 마일드해짐.

### Step 3. 시스템의 R&D 제안 리포트 출력
> **[대체 시뮬레이션 리포트]**
> * **신규 개발 요리명**: "안심소화 건비 삼계탕"
> * **개발 방향**: 체력 보강 중심에서 **"예민한 장 및 영양 흡수력 개선"** 중심으로 기능성이 리포지셔닝되었습니다.
> * **추가 추천 조치**: 산약의 흡수를 돕고 비위를 따뜻하게 보완하기 위해, `건강(말린 생강)` 분말 5g을 추가 조리 가이드에 넣으시면 맛의 풍미와 기능성이 극대화됩니다.

## 5. 특허 청구항 연계 전략 (Intellectual Property)

대체 시뮬레이터의 논리 구조는 차후 특허 포트폴리오를 구성할 때 독점적 방어막이 됩니다.

* **청구 범위 키워드**:
  1. 기 설정된 레시피 데이터베이스로부터 특정 요리를 선택하여 입력하는 모듈.
  2. 선택된 요리에 포함된 기준 식재료를 사용자가 임의의 대체 후보 재료로 변경하는 UI 모듈.
  3. 대체 후보 재료의 한의학적 기미 속성 및 표준 기능성 코드를 온톨로지 지식 베이스로부터 조회하여, 대체 전후의 다차원 생리 활성 변화 및 미각 조화성을 시뮬레이션 계산하는 엔진 모듈.
  4. 시뮬레이션된 결과를 다차원 레이더 차트 및 텍스트 리포트로 출력하여 신규 메뉴 발명을 가이드하는 플랫폼 시스템.
