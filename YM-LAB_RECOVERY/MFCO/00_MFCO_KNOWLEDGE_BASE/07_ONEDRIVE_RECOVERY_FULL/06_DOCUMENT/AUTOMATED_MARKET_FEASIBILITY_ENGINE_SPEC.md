# 📈 시장 타당성 자동 검증 엔진(AMFE) 상세 아키텍처 정의서
## (Automated Market Feasibility Engine Specification for Go/No-Go Decision)

본 정의서는 기획된 약선 메뉴 및 차(Tea) 아이디어가 시장에서 성공할 수 있는지 빅데이터와 알고리즘을 기반으로 검증하여, 시장성 없는 아이디어의 무분별한 개발로 인한 리스크를 원천 차단하는 **"시장 타당성 자동 검증 엔진(AMFE)"**의 데이터 파이프라인 및 평가 공식(Viability Scoring)을 수립합니다.

---

## 1. 전역 데이터 검증 파이프라인 (Data Pipeline)

```
[아이디어 기획 단계] ➡️ 메뉴명/핵심 원료/타겟 효능 키워드 입력
                              │
                              ▼ (실시간 API 호출 및 데이터 크롤링)
    ┌─────────────────────────────────────────────────────────┐
    │          4대 시장성 데이터 소스 실시간 수집             │
    │  1. 네이버/구글 월간 검색량 (소비자 실요구 규모)        │
    │  2. 소셜 버즈량 & 감성 지수 (트렌드 모멘텀)             │
    │  3. 이커머스 등록 상품 수 (경쟁 포화도)                 │
    │  4. 식약처 규제 및 특허청 DB (법적/기술적 장벽)         │
    └─────────────────────────────────────────────────────────┘
                              │
                              ▼ (AMFE 스코어링 엔진 작동)
        [시장성 종합 점수 (Market Viability Score) 산출]
                              │
         ┌────────────────────┼────────────────────┐
         ▼ (Score >= 75)      ▼ (50 <= Score < 75) ▼ (Score < 50)
           [GREEN (Go)]        [YELLOW (Pivot)]     [RED (No-Go)]
          즉시 상품화 추진       컨셉 보완/대체 제안      기획 즉시 기각
```

## 2. 4대 시장성 검증 빅데이터 소스

| 데이터 영역 | 수집 엔드포인트 / API | 분석 목적 | 추출 데이터 필드 |
| :--- | :--- | :--- | :--- |
| **소비자 실요구 (Demand)** | 네이버 검색광고 API, Google Trends API | 이 기획이 사람들이 실제로 해결하고 싶어서 스스로 검색하는 문제인지 검증 | 월간 검색량(PC/Mobile), 성별·연령별 비율, 계절성 지수 |
| **시장 모멘텀 (Momentum)** | Instagram Graph API, 블로그 크롤러 | 소셜 미디어 상에서 트렌드로 부상하고 있는 주제인지 파악 | 해시태그 언급 빈도(버즈량), 전월 대비 증가율, 긍/부정 감성 단어 비율 |
| **경쟁 포화도 (Competition)** | 네이버 쇼핑 API, 배달플랫폼 카테고리 데이터 | 이미 시장에 유사 상품이 포화 상태여서 가격 경쟁이 심한 레드오션인지 판단 | 등록 상품 수, 평균 가격대, 상위 노출 판매처 수 |
| **규제 및 장벽 (Barrier)** | 식품안전나라 원료 DB, 특허청(KIPRIS) OpenAPI | 법적인 판매 불법 여부 및 타사 특허 침해 여부 필터링 | 식용 불가 원료 포함 여부, 유사 특허 매칭률 |

## 3. 시장 타당성 종합 평가 공식 (Market Viability Scoring)

엔진은 수집된 원시 데이터를 활용하여 **0~100점 사이의 최종 시장성 점수(Viability Score)**를 계산합니다.

$$\text{Viability Score} = \left( w_d \cdot \text{Demand} + w_m \cdot \text{Momentum} \right) \times \left( 1.0 - \text{CompPenalty} \right) - \text{BarrierPenalty}$$

### 1) 수요 점수 (Demand Score: 0 ~ 50점)
* 타겟 키워드의 월간 검색 총량($V$)에 따라 로그 스케일로 점수 부여. (검색량이 아예 없으면 기본 점수 급감)
$$\text{Demand} = \min\left(50, 10 \cdot \log_{10}(V)\right) \quad (\text{단, } V \ge 100)$$

### 2) 모멘텀 점수 (Momentum Score: 0 ~ 50점)
* 최근 3개월간 소셜 버즈 증가율($R$) 및 긍정 감성 비율($S$)의 가중합.
$$\text{Momentum} = 50 \times \left( 0.6 \cdot \frac{R}{1.0 + R} + 0.4 \cdot S \right)$$

### 3) 경쟁 패널티 (Competition Penalty: 0.0 ~ 0.5)
* 등록된 경쟁 상품 수($N$) 대비 월간 검색량($V$)의 비율로 포화도를 계산합니다.
$$\text{CompPenalty} = \min\left(0.5, \frac{N}{V} \times 10\right)$$
* *의미*: 검색량 대비 상품 수가 너무 많으면 경쟁 패널티가 커져 종합 점수가 깎입니다.

### 4) 장벽 패널티 (Barrier Penalty: 0 또는 100점 감점)
* 식약처 고시상 식용 불가 원료가 검출되면 **100점 즉시 감점 (기획 자동 드롭)**.
* KIPRIS 유사 특허 매칭률이 85% 이상일 경우 **30점 감점 (소송 리스크)**.

## 4. 의사결정 리포트 및 피드백 시나리오

개발자가 **"눈 피로 개선용 결명자 꽃차"** 아이디어를 입력했을 때의 시뮬레이션 결과 화면 예시입니다.

### [AMFE 타당성 분석 결과 리포트]
* **기획 아이디어**: 눈 피로 개선 및 상열감 완화용 결명자 국화 블렌딩 꽃차
* **종합 시장성 점수 (Viability Score)**: **78점 [GREEN - Go]**
  * **수요 (42/50)**: '안구건조', '눈 피로 차' 관련 월간 총 검색량 85,000건으로 고수요층 확인.
  * **모멘텀 (38/50)**: 인스타그램 내 '눈건강피로', '이너뷰티 꽃차' 해시태그 증가율 전월 대비 28% 상승.
  * **경쟁 패널티 (-5%)**: 동일 레시피(결명자+국화)로 등록된 쇼핑몰 상품 수가 12개에 불과하여 경쟁 강도 매우 낮음 (블루오션).
  * **장벽 검토 (Pass)**: 원료(결명자, 감국) 식품 공전 등록 완료 확인. 유사 특허 매칭률 24%로 특허 침해 우려 없음.

> "본 아이디어는 소비자 수요가 매우 크고 경쟁 상품이 적어 시장성이 극히 우수합니다. 
> 스마트폰 카메라 진단 후 눈이 피로한 고객에게 추천하는 리추얼 세트 상품(스팀 안대 패키지)으로 가맹점 런칭 시 높은 수익성이 기대되므로 즉시 제품화 진행을 권장합니다."

## 5. 특허 보호 장벽 (BM Claims)

* **독점 특허 청구 범위**:
  1. 기 설정된 약선 레시피 기획 데이터를 수신하는 단계.
  2. 수신된 기획 데이터의 매핑 키워드를 추출하여 웹 트렌드 데이터베이스, 이커머스 상품 등록 서버, 국가 규제 서버의 실시간 OpenAPI를 통해 관련 시장 데이터를 수집하는 단계.
  3. 수집된 소비자 검색 볼륨, 소셜 미디어 증가 모멘텀, 검색량 대비 상품 수의 비율로 계산되는 경쟁 포화도, 특허 및 식재료 허용 규제 장벽 점수를 종합하여 0~100점 사이의 시장성 스코어(Viability Score)를 연산하는 단계.
  4. 스코어가 설정값 미만일 때 해당 기획을 반려(No-Go) 처리하고 대안 재료를 제시하며, 설정값 이상일 때 가맹점 출시 메뉴로 확정하여 원스톱 스케일링하는 단계를 포함하는 인공지능 기반 약선 요리 R&D 시장성 자동 검증 플랫폼 제공 방법.
