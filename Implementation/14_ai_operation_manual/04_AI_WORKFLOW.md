## 1. Overview

인간 프로젝트 리더(Human Project Lead)의 지휘 하에 각 단계별 확장 가능한 **핵심 AI 역할(Core Roles)**들이 유기적으로 연동되어 작동하는 일련의 개발 워크플로우를 정의한다.

---

## 2. Role Transition Cycle

```
           ┌─────────────────────────────┐
           │     Human Project Lead      │ (요구사항 정의 및 최종 릴리즈 승인)
           └──────────────┬──────────────┘
                          │
                          ▼
           ┌─────────────────────────────┐
           │       Architecture AI       │ (모듈 설계 및 의사결정서 작성 보조)
           └──────────────┬──────────────┘
                          │
                          ▼
           ┌─────────────────────────────┐
           │      Implementation AI      │ (기능 구현 및 단위 테스트 작성 보조)
           └──────────────┬──────────────┘
                          │
                          ▼
           ┌─────────────────────────────┐
           │          Review AI          │ (코딩 표준 및 아키텍처 동결 검사 보조)
           └──────────────┬──────────────┘
                          │
                          ▼
           ┌─────────────────────────────┐
           │            QA AI            │ (verify 스크립트 실행 및 에러 분석 보조)
           └──────────────┬──────────────┘
                          │
                          ▼
           ┌─────────────────────────────┐
           │      Documentation AI       │ (README 및 마스터 보고서 정리 보조)
           └─────────────────────────────┘
```

## 3. Detailed Workflow Steps

### Step 1: 설계 (Architecture AI)
- **활동**: 인간 프로젝트 리더가 요구사항을 정의하면, Architecture AI가 파일의 상대 경로 및 클래스/함수 인터페이스 골격을 설계한다.
- **통제**: 기존의 Read-Only 파일과의 종속성 충돌 유무를 검사한다.

### Step 2: 구현 (Implementation AI)
- **활동**: 설계 규격을 전달받아 Implementation AI가 파이썬 코드를 실제 구현하고 단위 테스트 코드를 작성한다.
- **반영**: 인간 프로젝트 리더가 제안받은 코드를 로컬 소스 파일에 삽입한다.

### Step 3: 리뷰 (Review AI)
- **활동**: 구현된 코드 파일에 대해 Review AI가 파이썬 코딩 스타일 가이드(PEP8) 준수 여부 및 의존성 관계의 적합성을 사전 리뷰한다.

### Step 4: 검증 (QA AI)
- **활동**: 로컬 터미널에서 `python YM-LAB_RECOVERY/scripts/verify_*.py`를 실행하여 정합성을 최종 검사한다.
- **피드백**: 검증 실패(FAIL) 발생 시, QA AI가 터미널의 에러 코드를 분석하여 자가 치유(Self-Healing)를 위한 버그 수정본을 제시한다.

### Step 5: 문서 (Documentation AI)
- **활동**: 기능이 모두 검증되면 Documentation AI가 `README.md` 및 `master_report.md` 등의 산출물을 명명 규칙과 문서 구조 양식에 맞추어 보강한다.
- **병합**: 인간 프로젝트 리더가 수동 최종 검수 후 Git 커밋 및 PR을 오픈한다.
