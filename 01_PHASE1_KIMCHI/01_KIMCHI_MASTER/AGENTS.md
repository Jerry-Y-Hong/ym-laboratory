1. Analyze
2. Plan

# PROJECT RULES

- Preserve the existing folder structure.
- Do not rename folders or files unless instructed.
- Do not create duplicate content.
- Reuse existing assets whenever possible.
- Follow the project's naming conventions.
- Keep outputs modular and reusable.

---

# FILE MODIFICATION RULES

Before modifying existing files:

- Explain what will change.
- Wait for approval.
- Modify only approved files.

Never overwrite files without permission.

---

# CONTENT RULES

Structure comes before content.

Create templates before populating data.

Maintain consistency across all MASTER databases.

---

# QUALITY ASSURANCE

Before completing any task, verify:

- Correct folder
- Correct file name
- Correct structure
- No duplicate content
- Consistent formatting

---

# PRIORITY

When instructions conflict, follow this order:

1. User instructions
2. AGENTS.md
3. Project Guide documents
