# 구현 계획서: MFCO 바이오 헬스케어 R&D 에코시스템 시뮬레이터 구축

본 구현 계획서는 웹 사이트 프론트엔드(`mfco-website`)를 실제 프로젝트 데이터베이스 파일(`02_CORE_DB`, `04_MAPPING_ENGINE`) 및 파이썬 추론 엔진의 핵심 로직과 연동하여 **MFCO 바이오 헬스케어 R&D 에코시스템 시뮬레이터**로 고도화하기 위한 상세 작업 계획입니다.

이를 통해 단순 화면 목업 수준이었던 사이트를 약선 요리 개발, 피부/다이어트용 처방 보조식품 및 외용제 설계, SCI급 생의학 연구 문헌 연계, 글로벌 교육 포럼을 제공하는 고정밀 플랫폼 시뮬레이터로 전환합니다.

## 제안된 변경 사항

### 1. 코어 엔진 및 추론 파이프라인 이식

#### [NEW] [mfcoInference.js](file:///c:/Users/car13/OneDrive/MFCO_MASTER_RECOVERY/mfco-website/src/utils/mfcoInference.js)
파이썬 엔진(`mfco_inference_engine.py`)의 다층 의미론적 추론 공식과 5반상 조합 로직을 JavaScript로 그대로 포팅합니다.
* **근본 원인 계산**: 18가지 증상(ST) 입력을 5대 근본 원인(RC01~RC05) 점수로 변환.
* **체질 보정 가중치 적용 (`γ = 0.4`)**: 소음인 등의 체질 취약 증상 매칭 시 가중치 보정.
* **장부 보정 가중치 적용 (`δ = 0.5`)**: 오장육부(간/심/비/폐/신) 약화 정보 연동 시 원인 점수 보정.
* **절기 공명 가중치 적용 (`β = 0.3`)**: 입력된 날짜(또는 현재 날짜)에 따른 24절기 계절성 보정.
* **레시피 추천 점수 산정 공식**:
  * **제형-효능 시너지 (Efficacy-Form Synergy)**: 생체/원물(`FORM_RAW`) 사용 시 아로마/신경안정 기능(`SF017`, `SF008`) 가산점 및 비수용성 성분 감점; 분말(`FORM_POWDER`) 사용 시 비위 강화(`SF022`) 가산점; 농축액(`FORM_EXTRACT`) 사용 시 수용성 다당류 보강 가산점 적용.
  * **소화기 상태 보정**: 소화불량(`ST-005`) 등 감지 시 소화가 쉬운 미음/죽류 및 차류 강력 가산점, 무거운 육류 식재료 감점 적용.
  * **영양학적 안전 가드레일**: 소화기 이상 환자 식단 구성 시 나트륨 800mg 초과 또는 지방 5g 초과 요리에 감점 패널티 적용.
* **다층 반상 식단 조립기 (Dynamic Bansang Composer)**: 추천된 메인 레시피를 기반으로 주식(Staple), 국/탕(Soup), 김치(Kimchi), 찌개/조치(Stew), 종지/소스(Sauce) 외에 **사용자가 설정한 첩 수(3첩/5첩/7첩/9첩/12첩)**에 맞춰 동적으로 사이드 반찬을 추출/조합하고 합산 영양 성분을 계산.

  2. **피부 및 다이어트 처방 보조제/외용제(Topical) 개발 캔버스**: 타겟 목적(피부 활성 `ST-006` 또는 다이어트/체중조절 `ST-016`)을 선택하고, 병풀(Centella), 황기, 의이인 등의 핵심 기능성 한약재를 조합한 뒤, 제형(외용고, 에센스, 캡슐보조제)을 지정하여 약성 시너지 계산, 조제 지침, 부작용 및 안전성 권고사항을 생성하는 캔버스 시뮬레이터 구축.

## 검증 및 테스트 계획

1. **추론 점수 정확성 검증**:
   * 자바스크립트 엔진의 계산 결과가 파이썬 엔진 시뮬레이션 결과(`scratch/inference_multilingual_preview.txt`)와 완벽히 일치하는지 임의의 케이스로 검증합니다.

   * PubMed, PubChem 등 실제 외부 API 호출 시 CORS 에러나 시간 초과를 예방하기 위해, 클라이언트 단에서 비동기 호출 성공 시의 데이터를 렌더링하고 실패 시에도 모의 SCI 백업 데이터를 우아하게 제공하도록 예외 처리합니다.
3. **사용자 사용성 검증**:
   * 로컬 개발 서버(`npm run dev`)를 실행하여 각 페이지(진단기, R&D 캔버스, 외용제 설계기, 창업 메뉴 검색기, 글로벌 포럼 강좌)의 동작과 애니메이션 피드백을 수동 검증합니다.
